# LLM Mathematical Reasoning

This project implements code for teaching large language models (LLMs) mathematical reasoning, with the main technique being Supervised Finetuning (SFT) to enable the model to learn responses generated by DeepSeek-R1, thereby learning complex mathematical reasoning skills (such as reflection, verification, etc.).

This experiment is based on the [DeepMath-103K](https://huggingface.co/datasets/zwhe99/DeepMath-103K) dataset and the [Qwen/Qwen2.5-Math-1.5B](https://huggingface.co/Qwen/Qwen2.5-Math-1.5B) model, guiding readers through a preliminary attempt at the complete pipeline of data distillation on mathematical tasks.

## Experiment Objectives

- Understand data cleaning and preprocessing for mathematical datasets
- Understand the basic methods for fine-tuning large language models
- Understand the basic methods for inference using large language models
- Understand the basic methods for evaluating mathematical capabilities of models

## Environment Requirements

- Python 3.8+
- PyTorch
- Transformers
- Datasets
- vllm
- GPU with at least 40GB VRAM

## Experiment Documentation

1. Open the Jupyter notebook:
```bash
jupyter notebook sft_math.ipynb
```

2. The notebook contains several main sections:
   - Dataset download and preprocessing
   - Model loading and training
   - Model generation and evaluation

3. For specific operations and instructions, please refer to the notebook.

## Output Files

The training process will generate checkpoint files from training, and the generation results will be saved after generation is complete.

## Notes

- The model and data need to be loaded from Hugging Face. For domestic users, you can use the https://hf-mirror.com mirror to download.
- Ensure you have sufficient disk space to save the original model files, data files, and trained checkpoints (reserve at least 50GB space)
- A GPU with at least 40GB VRAM is required
